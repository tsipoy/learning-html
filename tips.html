Tips on how to write html

How to write html quicker

If you want to write html faster you can follow these instructions.

  1. First, you should respect all the rules in html such as attribute and nesting.If you follow the rules, you may not get stuck writing your code.
  2. Learn a different kind of short cut and apply it when you write your code, which mean use a lot your keyboard instead of using mouse. It is really helpfull if you want to be faster, but it should be in the right way.
  3. Make sure that your code is well written so, when you run it through the html5 validator, you will not spend much time to fix errors or even do it again.
  4. When you start to write your code, it is better to set up your indenting.It will be tiring and just wasting time if fix it after your long html code.

Tools you need to write html

  1. Define a document type on the top of your code as <!DOCTYPE html>.It is not an html element but it is a required preamble found.
  2. Use html elements to form your html. Thera are a lots of elements that we can use. Also, we can use a different type of elements such as form element, head elements and table elements to create our sematic html.The element: The opening tag, the closing tag and the content together comprise the element.
  • The opening tag: this consists of the name of the element (eg:p), wrapped in opening and closing angle brackets. The element begins or starts to take effect where the paragraph begins.
  • The closing tag: this is the same as the opening tag, except that it includes a forward slash before the element name. Failing to add a closing tag is one of the standard beginner errors and can lead to strange results.

How to make sure your work is high-quality

  1. Use whitspace as indentations or line breaks, they are really helpfull to make high quality code. Your code will be really messy and difficult to undrstand if you forgot to use it.
  2. Check your code to the html5 validator. It help you fix your unexpected error.
  3. Try to not make the hard-coded, your code is always cool even if it is simple but it should be smarted.
